## Day 9 总结

主要是在「Model Context Protocol (MCP)」的深入应用与「Agent-to-Agent (A2A)」智能体协作协议的实战上。从零构建本地 MCP 服务（文件管理、知识库），进阶到使用 Qwen-Agent 链接本地与远程（高德、Bing、Web Fetch）的 MCP 工具，最后实现了基于标准化协议的 Agent 互联互通（服务发现与任务协作），打通了智能体之间的“社交网络”。

### 记忆卡片
- **MCP (Model Context Protocol)**：一种开放标准协议，用于连接 AI 模型与外部数据/工具。它将“工具”标准化为服务，使得 Agent 可以像插拔 USB 设备一样挂载各种能力。
- **FastMCP**：`mcp` 库提供的高级抽象，能够通过简单的装饰器（`@mcp.tool()`）快速将 Python 函数封装为 MCP 服务。
- **Hybrid MCP (混合工具链)**：Agent 可以同时挂载本地运行的 MCP 服务（如隐私数据处理）和远程托管的 MCP 服务（如 ModelScope 上的搜索服务），实现能力互补。
- **Agent-to-Agent (A2A)**：智能体之间的通信协议。核心机制包括“服务发现”（通过 `/.well-known/agent.json` 暴露能力名片）和“任务提交”（标准化的请求与响应结构），让 Agent 能够自主寻找并调用其他 Agent。

### 理解
- **MCP 服务端开发 (`1.MCP`)**：
  - `1.桌面TXT文件助手.py`：基于 `FastMCP` 构建的**文件系统操作**服务。提供了统计、列出和读取桌面 `.txt` 文件的能力，演示了如何安全地暴露本地资源给 AI。
  - `2.开发规范查询助手.py`：基于**内存数据**的知识库服务。将多语言（Python, Go, Rust 等）开发规范硬编码在内存字典中，演示了如何构建轻量级的查询服务，零依赖且响应极快。

- **Qwen-Agent 调用本地 MCP (`2.Agent_Use_MCP`)**：
  - `1.文本统计智能助手.py`：作为 Client 端，通过 `mcpServers` 配置连接 `桌面TXT文件助手`。Agent 负责理解用户意图（如“统计字数”），自动调度 MCP 工具获取精确数据，体现了“大模型大脑 + 精准工具手脚”的协作模式。
  - `2.开发规范智能助手.py`：连接 `开发规范查询助手`，作为专业的编程助手，能够回答关于代码风格和最佳实践的问题。

- **调用公共与远程 MCP (`3.Agent_Use_Public_MCP`)**：
  - `1.高德地图智能助手.py`：**调用官方 Node.js MCP**。通过 `npx` 动态运行高德官方提供的 `@amap/amap-maps-mcp-server`，获取实时的 POI 搜索和路径规划能力，无需自己编写地图 API 调用代码。
  - `2.全能智能生活助手.py`：**混合架构 (Hybrid)** 的集大成者。同时挂载了：
    - **本地**：高德地图 (Node.js) - 处理地理位置。
    - **远程**：ModelScope 上的 **Bing Search** 和 **Web Fetch** (SSE 协议) - 处理联网搜索和网页阅读。
    - 展示了 Agent 如何灵活调度多源工具来解决复杂的生活问题（如“查路线”+“搜新闻”）。

- **Agent-to-Agent 协作 (`4.Agent_To_Agent`)**：
  - `1.A2A天气代理服务.py`：**服务提供者 (Server)**。使用 `FastAPI` 实现了符合 A2A 协议的微服务。通过 `/.well-known/agent.json` 暴露“名片”，定义了输入 Schema 和认证方式，并提供模拟的天气查询接口。
  - `2.A2A篮球活动决策代理.py`：**服务消费者 (Client)**。它不直接具备天气查询能力，而是通过 A2A 协议**自动发现**并调用上述的天气 Agent，根据返回的天气状况（雨/晴）自动决策是否举办篮球活动，展示了机器间的自动化协作流程。
