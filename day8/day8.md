## 二、功能调用与智能Agent开发

### 1. Function Calling的作用

- 本质理解
- 核心价值

	- 扩展模型能力（实时数据、专业计算）
	- 建立交互桥梁（第三方服务）

- 典型应用

	- 实时天气/股价信息
	- 数学/设备控制

### 2. 扣子平台介绍

- 平台定位
- 核心功能

	- 多平台发布（WebSDK、API）
	- 资源管理（30,000资源点/月）
	- 模板商店（81.5K+模板）

- 开发模式

	- 低代码提示词定义
	- 插件化扩展（天气、搜索、图像理解等）

### 3. Function Calling的作用

- 参数提取

	- 自然语言中提取结构化参数（如&quot;北京明天天气&quot;→城市=北京，时间=明天）
	- 技术提升：大模型显著提高提取精度

- 实现机制

	- 工具注册
	- 动态调用
	- 典型工具：墨迹天气、头条搜索等

- 开发要点

	- 参数校验
	- 错误处理
	- 性能优化（响应时间）

### 4. Function Calling与MCP的区别

- MCP的定位

	- 全称：Model Context Protocol
	- 功能：标准化多模型接口，支持多轮、多模型调用

- 接口性质

	- Function Calling：模型私有接口
	- MCP：开放标准接口

- 开发成本

	- Function Calling：多模型适配成本高
	- MCP：一次开发支持多模型

- 适用场景

	- Function Calling：单次调用
	- MCP：复杂多轮、连续操作

- 未来展望

	- MCP将趋于主流
	- 协议不断优化

### 5. 天气调用功能讲解

- 扣子平台搭建

	- 创建智能体：天气助手
	- 命名规范：唯一性（最多20字符）
	- 功能描述：展示给用户
	- 创建方式：标准或AI创建

- MCP协议与插件

	- 提供API调用能力

	- 典型插件：墨迹天气、高德地图、图像生成

- 配置与测试

	- 时间处理：转换相对时间
	- 调试技巧：参数检查与时间插件

- 大模型的选择

	- 千问三优势：性价比高、部署灵活、协议友好
	- 适用场景：企业（32B）、移动（8B/4B）、轻量（1.7B）

- 架构特点
