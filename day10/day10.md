# 智能体设计与应用结构化知识树

## 一、智能体设计

### 学习目标

- 定位：架构设计角度开发智能体系统，结合实际案例
- 行业应用：证券行业为例，方法论迁移至医疗等场景

### Agent架构类型对比

- 反应式：即时响应，无复杂决策（私募基金问答）
- 深思熟虑式：深度分析推理（智能投研）
- 混合式：结合优势，平衡速度与决策（投顾AI）

### 技术实现要点

- 错误处理：

	- 透明化：真实错误暴露
	- 用户导向：配额错误明确指引

- 代码优化：

	- 移除冗余逻辑
	- 简化异常处理

- 性能优化：

	- 缓存(Token节省77.8%)
	- 严格类型检测

### 实践应用建议

- 行业适配：场景可替换，深思熟虑式架构注意应用
- 开发准备：提前下载课件，网页版工具优化中

## 三、智能体系统的设计与应用

### 智能体与大模型区别

- 大模型：提供基础能力，任务一次问答
- Agent：自主任务完成，多步调用
- 能力：

	- 任务分解
	- 记忆：短期、长期
	- 外部API连接

- 商业价值：Token消耗高（百万级）

### Generative Agents

- 概念：模拟人类行为
- 发展：Google 与 Stanford 提出
- 关键能力：

	- 自主移动
	- 环境交互
	- 社交传播
	- 关系记忆

- 系统架构：

	- 居民小镇(25Agent)
	- 场景：咖啡馆、学校等

- 开发工具：

	- 角色定义（职业、关系）
	- 自然语言行为
	- LangChain框架

- 趋势：应用从问答转向Agent

### Al Agent工具对比

- 结构：

	- LangChain(链式)
	- LangGraph(图结构)
	- Qwen-Agent(多模态)
	- Coze(可视化)
	- Dify(API优先)

- 企业偏好：

	- 外企：LangChain、LangGraph
	- 国内：Coze、Dify

- 开发方式：

	- 高代码：需编程
	- 低代码：拖拽配置
	- 趋势：混合发展

- 框架：LangChain1.0升级，LangGraph支持动态
- 工具：Chained、Graph结构、多模态支持

### 智能体设计流程

- 核心原则：

	- 简洁：核心工具隔离
	- 透明：显示决策要素
	- 功能：模块完整，包括测试

- 框架选择：从高阶到底层逐步优化
- 客户需求：响应速度、响应策略优先
- 意图识别：

	- 对话管理
	- 长文本分块

- 机制：

	- 感知-决策-执行
	- 闭环流程

- 应用场景：

	- 扫地机器人、游戏、安防

- 关键技术：

	- 模型验证
	- 轨迹日志分析

##  其它总结

### 一、智能体设计与实现

- 1. 学习目标

	- 定位

		- 从架构设计角度探讨智能体系统开发，通过实际案例进行应用分析。

	- 行业应用

		- 以证券行业为例，方法论可迁移至医疗等行业。

	- Agent架构类型对比

		- 反应式 (Reactive): 即时响应，无复杂决策，如私募基金指引问答助手。
		- 深思熟虑式 (Deliberative): 需分析推理，用于智能投研助手。
		- 混合式 (Hybrid): 结合响应速度与决策质量，应用于投顾AI助手。

- 2. 技术实现要点

	- 错误处理原则

		- 透明化：真实暴露错误，避免隐藏。
		- 用户友好：提供明确解决方案。

	- 代码优化

		- 移除冗余逻辑，简化异常处理。

	- 性能优化

		- 使用缓存减少Token消耗（示例节省77.8%）。
		- 类型检查确保数据安全。

	- 实践应用建议

		- 行业场景替换建议，医疗重点关注深思熟虑架构。
		- 课件资料提前下载，网页版工具持续完善。

- 3. 互动答疑

	- 知识库管理

		- 文档数量异常：核实索引和格式。
		- 多文档检索：建立全部索引，确保覆盖所有内容。

	- 新闻Agent优势与限制

		- 优势： 灵活架构、简洁代码、便捷管理。
		- 局限： 仅支持单步规划，复杂场景需结合其他工具。

	- 开发模型选择

		- 结构化查询、图文分析、完整工作流、模型微调。
		- 选型以业务场景为导向，如金融交易优先Agent架构。

	- 数据交换&amp;虚拟机申请

		- MCP部署：中心服务器或内网适配层。
		- 移动端方案：云虚拟机部署，私有化安全方案。

	- 大模型配置

		- 千问Code Plus：代码生成优良。
		- 高性能模型：千问3模型提升40%完成度。

	- 智能体设计技巧

		- 通过示范、微调、混合架构提升稳定性。
		- 让Agent自主判断，规则干预。

	- 常见模型分类

		- LLM：语言理解。
		- VLM：图文结合。
		- 多模态：跨模态信息融合。
		- 选择平台：企业推荐Dify、Coze；个人推荐Cursor。

- 4. 智能体系统的设计与应用

	- 1）智能体与大模型的区别

		- 大模型（LLM）: 提供基础能力，单次问答。
		- Agent: 自主任务系统，多步调用。
		- 能力对比: 规划、记忆（短期/长期）、工具调用。
		- 商业价值: Token消耗高，未来商业模式关注Token销售。

	- 2）生成式智能体（Generative Agents）

		- 概念：模拟人类行为的自主体。
		- 关键技术：行为模拟、空间交互、关系记忆。
		- 系统架构：Smallville模型，包含自主移动、环境交互和社交传播。
		- 开发工具：自然语言定义角色，LangChain支持模块化。

	- 3）Al Agent工具对比

		- 架构特性：链式、图结构、可视化、API优先。
		- 应用偏好：国外偏向LangChain，国内偏好Coze与Dify。
		- 高低代码：高代码（LangChain等）便于定制，低代码（Coze/Dify）快速部署。
		- 混合趋势：结合使用以提升效率。

- 5. 实践流程及最佳实践

	- 1）设计原则

		- 简洁性：功能模块化，避免重叠。
		- 透明性：白盒化决策过程。
		- 功能封装：完整开发流程，说明与测试。

	- 2）框架选择

		- 结合需求选用高阶框架（LangChain）或底层组件。
		- 根据业务调整模型和数据存储方案。

	- 3）意图识别与任务处理

		- 开发专用意图识别模块。
		- 分块处理长文本。

	- 4）反应式架构

		- 实现感知-决策-执行闭环。
		- 兼顾响应速度（如机器人避障）和长期规划（如投资策略）。

	- 5）智能体分类

		- 反应式智能体：快速对应紧急任务。
		- 深思熟虑智能体：多阶段长远任务。
		- 混合智能体：动态切换。

	- 6）工具集开发

		- 以功能独立、安全为目标。
		- 50%开发时间投入工具打造。
		- 设计权限控制制度。

	- 7）实践总结

		- 关注工具和提示词优化。
		- 持续迭代，客户项目2-3天打磨。

- 6. 内容总结

	- Agent模式应用广泛，实用性强。
	- 高代码开发更受偏好，低代码适合快速搭建。
	- 重点在工具选择和提示词设计。
	- 持续优化和迭代达到高性能。

- 7. 构建Agent的核心思想

	- 1）Agent的局限性

		- 上下文限制（10-20k Token）。
		- 视角模拟缺陷。
		- 任务成本评估。

	- 2）优化方法

		- 模型验证、日志分析。
		- 从环境、工具、提示层面优化。

	- 3）工作模式

		- 速度优先、质量优先、混合。
		- 通常使用图结构（LongGraph）可视化。

	- 4）开发建议

		- 使用五彩工具生成HTML报告。
		- 先实现基础模型，再迁移到场景。
		- 持续站在Agent角度优化。

### 三、深思熟虑智能体架构

- 1. 投研报告流程

	-  

		- 五阶段：感知→建模→推理→决策→报告。
		- 状态管理：ResearchAgentState保持信息。

	- 2. 关键技术

		- 模块化：定义各阶段函数。
		- 错误检测：内置异常处理。
		- 框架设计：LangGraph结构（节点、边、状态管理）。

	- 3. 核心节点设计

		- 感知：市场数据。
		- 建模：行业状态。
		- 推理：方案生成。
		- 决策：选择最佳方案。
		- 报告：输出分析。

- 4. 高低代码对比

	- 高代码：约500行Python。
	- 低代码：图形化拖拽平台。
	- 开发效率：AI辅助比人工快。
	- 适用场景：复杂（高代码）vs 简单（低代码）。

- 5. 关键技术

	- 角色设定：每个节点具有专业角色。
	- 置信度评分：优先方案。
	- 状态流转：使用TypedDict。
	- 错误处理：定义error字段。
	- 提示设计：变量模板。

- 6. 分段与总结

	- 大文档处理：分块提取，信息压缩。
	- 调试经验：错误诊断、版本迭代。
	- 限制：Token收费，长时记忆缺失。

- 7. AI对话中的开发作用

	- 作用：提高指令准确性，预判问题。
	- 无开发亦可：持续使用成长。
	- 方法：观察AI流程，调整策略。

- 8. 五彩Agent原理

	- 组件：工具集、任务规划器。
	- 异常：日志监控，系统指令。
	- 质量：提示、API、流程反复调试。
	- 防幻觉：双Agent校验。
	- 大文件处理：分段总结。
	- 调试：人工监督、异常追踪。

- 9. 代码逻辑与HTML生成

	- 核心流程：数据→模型→推理→决策→报告。
	- 技术：LangGraph，JSON格式输出。
	- 工具：文件工具，任务管理。
	- 输出：结构化报告，异常捕获。
