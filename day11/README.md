## Day 11 总结

主要是在「多模态大模型 (Multimodal LLM)」的实战应用上。深入探索了 **视觉-语言模型 (Vision-Language Models)** 在保险、视频分析和文档处理三大场景的应用。从车险定损的批量图片分析，到交通事故的视频损伤检测，再到复杂 PDF 文档的智能解析，展示了多模态 AI 在企业级应用中的强大能力。

### 记忆卡片
- **VLM (Vision-Language Model)**：能够同时理解图像和文本的多模态大模型，如 Qwen-VL、InternVL。支持图像内容识别、视觉问答 (VQA)、物体定位 (Grounding) 等任务。
- **Grounding (视觉定位)**：模型不仅能识别图像中的物体，还能返回物体的边界框坐标，实现"框出图中XX"的精确定位能力。
- **动态视觉分块 (Dynamic Visual Preprocessing)**：InternVL 特有的技术，根据图像宽高比自适应切割为多个高分辨率小块，兼顾全局感知与局部细节识别。
- **多模型轮询 (Model Rotation)**：当某个模型 Token 额度耗尽时，自动切换到下一个可用模型的容错机制，最大化利用免费试用额度。
- **MinerU**：开源的文档智能解析工具，支持 PDF、图片等格式的深度内容提取，包括文本、表格、公式等。

### 理解
- **VL 模型在保险行业的应用 (`1.VL模型在保险行业的应用`)**：
  - `1.保险_VLM_批量图片分析.py`：基于 **Qwen-VL-Max** 的批量处理系统。从 Excel 读取理赔指令和图片标识符，调用阿里云 API 进行多模态推理，将分析结果自动回写到新 Excel 文件。支持单图/多图处理，内置异常捕获机制。
  - `2.通义VL_本地图片内容识别.py`：使用 **DashScope 原生 SDK** 调用 Qwen-VL-Plus 模型识别本地图片内容，演示了基础的多模态对话流程。
  - `3.通义VL_多轮对话与定位检测演示.py`：全面演示 Qwen-VL 的核心能力，包括云端 API 调用、多轮对话上下文保持、视觉定位（返回物体坐标），以及本地 GPU 环境下的自动边界框绘制。

- **VL 模型对视频的应用 (`2.VL模型对视频的应用`)**：
  - `1.InternVL2.5_视频损伤分析与多轮对话.py`：基于 **InternVL 2.5 (InternVideo2_5_Chat_8B)** 的本地视频分析方案。实现了智能模型生命周期管理（自动检测/下载）、动态视觉分块技术、自适应时空采样策略（均匀分段提取关键帧）、深度多轮上下文记忆（支持中英混合提问），以及 BFloat16 低显存推理优化。
  - `2.QwenVL_视频损伤分析与多轮对话.py`：基于 **DashScope API (qwen-omni-turbo-latest)** 的云端视频分析方案。无需本地 GPU，直接上传视频文件进行多轮对话，展示了云端原生视频理解的极简部署方式。
  - `3.QwenVL_多模型轮询_视频损伤分析.py`：实现**多模型自动轮询机制**。维护一个模型优先级池（Turbo → Flash → 备选版本），当当前模型额度耗尽时自动切换到下一个可用模型，并通过上下文重建保持对话连贯性，最大化利用免费试用额度。

- **MinerU 在文档中的应用 (`3.MinerU在文档中的应用`)**：
  - `1.本地使用MinerU.py`：使用 **MinerU 命令行工具** 本地处理 PDF 文档。通过 `uv run mineru` 命令调用，支持实时日志输出和错误过滤，将 PDF 解析为结构化内容。
  - `2.API使用MinnerU.py`：通过 **MinerU API** 进行文档提取。提交任务后通过轮询获取处理进度，支持 OCR、公式提取等高级功能，适合云端集成场景。
